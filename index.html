AIを活用したLPコーディング自動化システム

卒業制作資料

制作したアウトプット

簡易的なLPコーディングのAIによる自動化システム

本プロジェクトでは、従来手作業で行っていたランディングページ（LP）のコーディング作業を、AIと自動化技術を組み合わせて効率化するシステムを開発しました。

対象LP: サブウェイ パーティートレイキャンペーンページ

  • URL: https://www.subway.co.jp/campaign/partytray/
  • 特徴: 商品画像中心の視覚的なレイアウト
  • 構成: ヒーローセクション、商品紹介、価格表示、店舗情報

アプローチの概要

初期設定したGOAL

「AIにプロンプトを送ることで、あとは自動でコーディングしてくれるシステム」
  • ユーザーが簡単な指示を与えるだけで完成したLPが生成される
  • デザインからコーディングまでの全工程を自動化
  • 非エンジニアでも簡単にLPを作成できる環境の構築

実装アプローチの変遷

Phase 1: Claude Desktop + Photoshop連携

MCPサーバーによる統合アプローチ

Claude Desktop ←→ MCPサーバー ←→ Photoshop

実現できた機能:

  • Photoshopでの画像書き出し自動化
  • AIによる画像素材の生成・編集指示
  • デザインファイルからの素材抽出

課題:

  • Claudeのコーディング出力の品質が不安定
  • 複雑なレイアウトの再現性に限界
  • デザインの一貫性を保つことが困難
Phase 2: ハイブリッドアプローチへの転換

実際に採用した2段階自動化システム

工程1: AI画像素材自動化

  • ツール: Claude Desktop + Photoshop (MCP連携)
  • 機能:
    • 商品画像の自動生成
    • バナー・アイコン素材の作成
    • 画像の一括書き出し・最適化

工程2: HTML/CSS半自動化

  • ツール: Claude Code + Node.js
  • 機能:
    • 定型的なHTMLテンプレートの生成
    • レスポンシブCSSの自動出力
    • 画像パスの自動設定
  • ソースコード: https://github.com/bbm-tnakajima/lp_auto_generate/blob/main/lp-generate.js

技術スタック

AIツール

  • Claude Desktop: 画像生成・編集の指示出し
  • Claude Code: コーディング支援・スクリプト作成
  • Photoshop (MCP連携): 画像素材の書き出し

開発環境

  • Node.js: 自動化スクリプトの実行環境
  • HTML/CSS: フロントエンド出力
  • MCPサーバー: AI・ツール間連携

自動化の実現方法

1. 画像素材の自動生成フロー

Claude Desktopへのプロンプト定義

以下の要件をpromptで指定：

  • 画像スライスの自動化
  • 画像書き出し先指定
  • 画像の書き出しファイルの命名規則を指定

実際のprompt例：

今開いている画像ファイルを適当に2分割してスライスして画像を書き出してください。
書き出し先は「/Users/t_nakajima/Desktop/images/subway/.」でお願いします。
書き出しファイルの命名規則は「image_{数字}」でお願いします。
フロー図: デザイン要件入力 → Claude Desktop → Photoshop MCP → 画像素材生成 → 自動書き出し

2. コーディング半自動化フロー

Node.jsスクリプトの機能概要:

  1. 画像の自動取得とコピー
    • デスクトップのimages/subwayフォルダから画像を取得
    • lp_contents/imagesにコピー後、最終的に出力先にコピー
  2. PC/SP両対応のHTML生成
    • 同じ画像からPC用とSP用の2つのHTMLを自動生成
    • レスポンシブ対応のCSSも同時出力
  3. YouTube動画埋め込み
    • 任意でYouTube動画を指定した位置に挿入可能
    • 動画IDの自動取得とembedコードの生成

実行方法:

# YouTube iframe を 1番目の contentsWrap に挿入
node mcp-server.js mycampaign 71tbHED8IvI 1

iframe を挿入せずに HTML 生成のみ
node mcp-server.js mycampaign

コマンド引数の説明:

  • mycampaign: キャンペーン名（出力フォルダ名）
  • 71tbHED8IvI: YouTube動画ID
  • 1: 挿入する位置（contentsWrapの番号）
フロー図: レイアウト要件 → Claude Code → Node.jsスクリプト → PC/SP HTML生成 → 画像パス自動設定 → YouTube埋め込み

課題と改善点

現状の制限

  • 完全自動化には至らず、人的介入が必要
  • 複雑なアニメーションやインタラクションは手動対応
  • デザインの創造性に制限

今後の改善方向

  • 3馬力生成AI（ChatGPT、Gemini、ClaudeをMCPサーバーで連携したもの）を使用してより高精度な自動化を実現
  • テンプレートの種類拡張
  • ユーザーインターフェースの改善

まとめ

当初の「完全自動化」という目標からは修正を余儀なくされましたが、実用的なハイブリッド自動化システムを構築することができました。

AIの現在の能力を正確に把握し、人間の創造性とAIの効率性を適切に組み合わせることで、実際のワークフローで活用できるツールを開発することができたと考えています。

今後の展望

  • 他のWebサイトタイプへの応用展開
